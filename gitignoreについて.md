📘 .gitignore 完全まとめ


🔵 .gitignoreとは？

Gitに「これは管理しなくていい」と伝えるファイル。



🔵 なぜ必要？

Gitは基本的に
フォルダ内のすべてのファイルを管理しようとします。

でも中には：

自動生成されるもの
再生成できるもの
不要なもの

があります。

それを無視するために .gitignore を使います。



🔵 今回の例

node_modules
*.js


🔹 node_modules

npm install で自動生成

何千ファイルもある

package.json があれば再生成できる

だから上げない。


🔹 *.js

tsc が自動生成

.ts があれば再生成できる

本体ではない

だから上げない。



🔵 本質

Gitに上げるのは

👉 人間が書いたものだけ

上げないのは

👉 機械が作るもの



🔵 仕組み

.gitignore に書くと：

git add . をしても無視される

git status に出てこない



🔥 一言まとめ

再生成できるものはGitに上げない。
